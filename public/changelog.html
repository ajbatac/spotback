SpotBack Changelog

All notable changes to this project are documented in this file.

[Unreleased]

  • Added: Added "Why SpotBack?" section to the login page for new users.
  • Added: Replaced login page icon with a custom PNG image (public/spotify.png).
  • Added: Programmatically generate app favicon via src/app/icon.tsx to resolve browser and build conflicts.
  • Changed: Updated login page description to be more energetic and engaging.
  • Changed: Restored hover animation to all primary action buttons for better user feedback.
  • Changed: Made the changelog link in the footer open in a new tab.
  • Fixed: Corrected page layout to ensure footer sits directly below content, not at the bottom of the viewport.
  • Fixed: Adjusted login page alignment to position content higher on the screen.
  • Fixed: Resolved favicon conflict error by removing the static file from /public and using the Next.js App Router's icon.tsx convention.

[0.2.0] - 2024-07-12

  • Added: QR code in the footer is now a clickable link to support the creator.
  • Changed: Improved UI consistency for all primary action buttons.
  • Changed: Increased size of the footer QR code for better visibility and usability.

[0.1.0] - 2024-07-11

  • Added: Initial Release of SpotBack!
  • Added: Secure Spotify OAuth 2.0 authentication.
  • Added: Fetches and displays user profile and all user playlists.
  • Added: Select one, multiple, or all playlists for backup.
  • Added: User-friendly dashboard to manage playlists and backups.
  • Added: Export selected playlists to JSON, XML, or a TXT file of URLs.
  • Added: Responsive landing page and dashboard.
  • Added: Header with user info and footer with project credits.
  • Added: Consistent UI theme with styled buttons, icons, and hover animations.
  • Added: Comprehensive README.md documentation and a public changelog.
